# File Status 文件状态


### 第1列状态说明：描述文件被添加、删除或其他修改

```objc
' ' 没有修改
'A' {
        被添加到本地代码仓库管理中, 但是还没有添加到服务器中
        服务器的仓库还未管理该文件
    }
'C' 冲突
'D'
    {
        被删除
        该文件已经在本地进行了删除, 但是还没有将删除的操作共享给服务器
    }
'I' 被忽略
'M'
    {
        被修改
        文件已经被管理, 并且服务器也有该文件,
        新修改的文件修改的内容没有提交到服务器
    }
'R' 被替换
'X' 外部定义创建的版本目录
'?' {
        文件没有被添加到本地版本库内
        svn 不识别该文件, 或者该文件被删除过,
        或者是给文件是新添加的文件, 还没有被本地代码仓库进行管理
    }
'!' 文件丢失或者不完整（不是通过svn命令删除的文件）
'~' 受控文件被其他文件阻隔

```
